# Architecture Decision Records

ADRs record material decisions without confusing a decision with its implementation or
validation.

Use an ADR for changes to the NSIT product contract, notation, identity model,
language adapters, trust boundaries, compatibility, verification, or release policy.

1. Copy `0000-template.md` to the next four-digit number and a short slug.
2. Open it as `Proposed` before or with implementation.
3. Record meaningful alternatives and falsifiable consequences.
4. Change it to `Accepted` only after approval.
5. Link implementation and evidence separately.
6. Supersede accepted ADRs with a new ADR; do not rewrite history.

Statuses: `Proposed`, `Accepted`, `Rejected`, `Deprecated`, `Superseded`.
